# クイックスタート: イベントデータソース更新

## 目的

自動テストを追加せずに、アプリが `yosakoi_event.csv` を読み込み、`event_id` でルーティングし、
AI データ注意書きと利用規約の免責文言を正しく表示することを確認する。

## 前提

- `frontend/app` の依存関係がインストール済みであること
- ローカル開発サーバーを起動できること

## 起動方法

```bash
cd frontend/app
npm ci
npm run dev
```

## 手動確認

### 1. 新しい CSV から一覧が表示される

- トップページを開く。
- 一覧が `yosakoi_event.csv` の内容をもとに表示されることを確認する。
- 画像、チーム数、最寄り駅、駐車場情報などの任意項目が空でもカード表示が崩れないことを確認する。
- `Status`、開催期間、開催地、説明などの必須項目が表示に反映されることを確認する。

### 2. 詳細ルートが `event_id` を使う

- 一覧から少なくとも 2 件のイベントカードを開く。
- URL パスが各レコードの CSV 上の `event_id` と一致することを確認する。
- 詳細ページを直接リロードしても同じレコードが解決されることを確認する。

### 3. 必須項目欠損の行が除外される

- 開発時に使うデータセットで必須項目のいずれかが欠ける行がある場合の UI 挙動を確認する。
- 正常行は表示され続け、不正行は一覧に出ないことを確認する。

### 4. 詳細ページの注意書きが見える

- 詳細ページで AI データに関する注意書きが表示されることを確認する。
- 注意書きの近くに公式 URL が表示され、押下できることを確認する。
- YouTube と地図が表示または利用でき、`updated_at` が画面に出ていないことを確認する。

### 5. 利用規約の免責文言が見える

- `/terms` を開く。
- データ精度に関する注意書きと損失免責の文言が含まれていることを確認する。

### 6. レスポンシブ確認

- モバイル幅: 一覧カードが自然に縦積みされ、詳細の注意書きが読みやすいことを確認する。
- デスクトップ幅: 詳細レイアウトの構造が保たれ、注意書きが埋もれないことを確認する。

## 想定結果

- アプリが新しい CSV スキーマを正しく読み取る。
- 詳細ルートが `event_id` で解決される。
- 有効なレコードでは必ず公式情報へ到達できる。
- 有効なレコードでは YouTube と地図を利用でき、`updated_at` は内部管理用のままである。
- データ精度と免責に関する文言が明確に見える。
