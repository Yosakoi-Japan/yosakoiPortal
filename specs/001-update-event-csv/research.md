# 調査: イベントデータソース更新

## 判断: `frontend/app/src/assets/data/yosakoi_event.csv` を唯一のイベントデータセットとして使う

**理由**:
- 提供された CSV には、`event_id`、`event_name`、`Status`、`official_url`、日付、位置情報、
  動画 URL、地図 URL、更新日などの必須列が定義される前提がある。
- ヘッダーは英語名なので、旧来の日本語ヘッダー前提ではなく新しい列構成に合わせて parser を揃える必要がある。
- `assets/data` 配下のローカルファイルをそのまま使えば、現行の静的読み込み構成を維持でき、新しい基盤も不要になる。

**検討した代替案**:
- `events.csv` を残して別変換する: 正式なソースを新 CSV に切り替える要件とずれるため不採用。
- リモート API 読み込みへ変更する: 単なるファイル差し替えと UI 修正を超えるため不採用。

## 判断: `event_id` を詳細ページのルートキー兼検索キーとして扱う

**理由**:
- feature spec では、詳細 URL に `event_id` を使うことが要求されている。
- 新 CSV は `event_id` を持っており、イベント名から ID を生成するより安定している。
- タイトルに空白、記号、重複名称があっても URL の曖昧さを避けられる。

**検討した代替案**:
- タイトル由来 ID を維持する: 壊れやすく、要求された契約にも合わないため不採用。
- 新しい slug を別途生成する: すでに `event_id` があり、仕様上も利用者に見える契約なので不採用。

## 判断: 必須列が欠ける行は除外し、正常行の表示は継続する

**理由**:
- 現在の spec では、`event_id`、`event_name`、`Status`、`official_url`、`start_date`、
  `end_date`、`location`、`description`、`youtube_url`、`latitude`、`longitude`、
  `map_url`、`updated_at` を必須として扱っている。
- 行単位で除外すれば、壊れた詳細ルートや空の公式リンクを防ぎつつ、正常データの UI 表示は維持できる。
- 現在の CSV では提示されている行に両項目があるため、この検証は停止条件ではなく保護策として機能する。

**検討した代替案**:
- 1 行でも不正なら一覧全体を失敗にする: 局所的なデータ不備でページ全体を悪化させるため不採用。
- 不正行も一覧に出し、詳細でだけ失敗させる: 防げる壊れ方を残すため不採用。

## 判断: 任意の補足項目は、あれば活用し、なければ自然に省略する

**理由**:
- CSV では `image_url`、`team_count`、`nearest_station`、`parking_info` のみが任意である。
- 補足列が空でも、一覧と詳細は読みやすく成立している必要がある。
- これは constitution の「シンプルな UX」と「モバイルでも破綻しない表示」に合致する。

**検討した代替案**:
- 任意項目が揃うまで詳細を出さない: 多くのレコードを不必要に隠すため不採用。
- 新 CSV の任意項目をすべて無視する: 画像や交通補足などの補完情報を活かせなくなるため不採用。

## 判断: 詳細ページに AI データ注意書きを追加し、利用規約にも免責文言を拡張する

**理由**:
- spec では、AI 収集データに誤りの可能性があることと、公式情報の確認を促す文言が必要である。
- 同じリスク前提は、サービス全体の免責として利用規約にも反映する必要がある。
- 詳細ページでは、公式 URL の近くに注意書きを置くことで、利用者の次の行動が明確になる。

**検討した代替案**:
- 利用規約だけに警告を書く: 利用者が Terms を見ずにイベント情報を使う可能性があるため不採用。
- 一覧だけに警告を書く: リスクが重要になるのは詳細確認時なので不採用。

## 判断: 検証は手動確認のみとする

**理由**:
- constitution で unit / integration / E2E テストの追加が明確に禁止されている。
- この機能は、ルート確認、一覧表示確認、注意書き確認、利用規約確認で十分に検証できる範囲である。

**検討した代替案**:
- parser の unit test を追加する: constitution 違反のため不採用。
- ルート確認のためにブラウザ自動化を入れる: constitution 違反のため不採用。
