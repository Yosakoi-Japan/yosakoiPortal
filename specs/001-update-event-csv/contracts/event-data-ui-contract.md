# 契約: イベントデータと UI 挙動

## 目的

`yosakoi_event.csv` の取り込み、UI へのイベント公開、一覧・詳細・免責表示の挙動について、
アプリケーション側の契約を定義する。

## データ取り込み契約

### 対象ファイル

- パス: `frontend/app/src/assets/data/yosakoi_event.csv`
- 形式: ヘッダー行ありの UTF-8 CSV

### 必須列

| 列名 | 用途 |
|--------|---------|
| `event_id` | ルートと検索の正規キー |
| `event_name` | 表示タイトル |
| `Status` | 公開状態または承認状態 |
| `official_url` | 公式確認先 URL |
| `start_date` | 開始日 |
| `end_date` | 終了日 |
| `location` | 開催地表示 |
| `description` | 概要文 |
| `youtube_url` | 動画参照先 |
| `latitude` | 緯度 |
| `longitude` | 経度 |
| `map_url` | 地図遷移先 URL |
| `updated_at` | 内部管理用のデータ更新日 |

### 任意列

`image_url`, `team_count`, `nearest_station`, `parking_info`

### バリデーション挙動

- 必須列のいずれかが欠ける行は除外する。
- 任意列が空でも、その行は有効とみなす。
- 詳細検索には、採用した行の `event_id` をそのまま使う。
- `note` カラムは参照しない。
- `updated_at` は内部管理用として保持し、利用者向け画面には表示しない。

## 一覧画面契約

### 入力

正規化済みのイベント一覧配列。

### 出力

- 有効行ごとに 1 枚のカードを描画する。
- カードには `title`、`area`、`description`、`period` を表示する。
- カード押下時は `/event/{event_id}` に遷移する。
- 不正行からはカードを生成しない。

## 詳細画面契約

### ルート

- パターン: `/event/{event_id}`

### 検索挙動

- ルートパラメータは `EventRecord.id` と照合する。
- 有効なレコードが存在する場合は詳細ページを描画する。
- 有効なレコードが存在しない場合は、一覧へ戻れるエラー状態を表示する。

### 詳細で必須の表示内容

- イベント名
- 開催地
- 開催日または開催期間
- 公開状態
- 公式 URL リンク
- YouTube リンクまたは埋め込み
- 地図表示または地図遷移リンク
- AI データ注意書き

### 詳細で任意の表示内容

- 参加チーム数
- 最寄り駅
- 駐車場情報
- 画像表示

## 免責表示契約

### 詳細ページの注意書き

各詳細ページには、少なくとも次と同等の意味を持つ文言を含めること。

- このイベントデータは AI 補助により収集されているため誤りを含む可能性がある。
- 最新かつ正確な情報は公式ページで確認する必要がある。

### 利用規約ページの免責文言

利用規約ページには、少なくとも次と同等の意味を持つ文言を含めること。

- 本サイトのデータは誤っている可能性がある。
- その誤りに起因する利用者の損失について、運営者は責任を負わない。

## レスポンシブ契約

- モバイルでは、注意書きと公式 URL が縦方向の自然な読書導線に入っていること。
- デスクトップでは、注意書きが低優先度情報の下に埋もれず視認できること。
- 一覧カードは、狭い画面幅でも横スクロールなしで読めること。
